# 拡張機能 更新信頼性向上プラン

## 現状の問題
- コードを修正しても、拡張機能に反映されない（更新ミス）が発生。
- 手順が煩雑で、ビルド忘れや読み込み忘れが起きやすい。
- 結果として確認作業の手戻りが多く、時間を浪費している。

## 解決策
確実な更新を行うための「鉄板フロー」と「自動化ツール」を導入する。

### 1. バージョン自動更新スクリプト (`update_extension.ps1`)
- `manifest.json` のバージョン番号を自動インクリメント（例: 1.0.0 → 1.0.1）
- `npm run build` を実行
- 完了時に変な音が鳴る（完了通知）
- **「拡張機能を再読み込みしてください」** という明確なメッセージを表示

### 2. 更新手順書の作成 (`EXTENSION_UPDATE_GUIDE.md`)
- 開発時とリリース時の手順を明確に区別。
- Chromeの「拡張機能を更新」ボタンを押すタイミングを明記。

### 3. ビルド確認機能
- 画面右上に現在のバージョン番号を常に表示し、更新されたか一目でわかるようにする。
